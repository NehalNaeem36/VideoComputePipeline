The UI should calculate ETA using progress and elapsed time.

Suggested ETA formula:

`eta_seconds = elapsed_seconds * ((1.0 - progress) / progress)`

The UI should support progress from either frame count or timestamp.

Preferred progress source:

Structured progress lines printed by `VideoComputePipeline.exe`.

If the pipeline does not currently print machine-readable progress lines, plan a small optional addition to the pipeline later:

`--progress-format kv`

Example line:

`PROGRESS frame=67392 total_frames=108000 timestamp_ms=1123000 duration_ms=1800000 fps=119.7 speed=2.0 detections=4821 output_bytes=1932735283`

The UI should parse lines beginning with `PROGRESS`.

If structured progress is not available yet, the UI should still show logs and elapsed running time.

5. Tooltips

Every important option should have a small tooltip using ImGui hover behavior.

Examples:

Decoder tooltip:

`CPU decodes video frames in system RAM. NVDEC uses NVIDIA hardware decoding and can keep frames on GPU. Use NVDEC for the high-throughput GPU path.`

Encoder tooltip:

`h264_nvenc uses NVIDIA hardware encoding. Use this for annotated video output with low CPU load.`

Confidence tooltip:

`Minimum detection confidence. Higher values reduce false detections but may miss weak objects. Typical range: 0.25 to 0.50.`

IoU tooltip:

`Controls non-maximum suppression. Lower values remove overlapping boxes more aggressively. Typical YOLO value: 0.45.`

Input size tooltip:

`Model input resolution. YOLOv5s commonly uses 640. Higher values may improve small-object detection but reduce FPS.`

Precision tooltip:

`fp32 is safest and most compatible. fp16 can be faster only if the TensorRT engine and GPU support it.`

Output format tooltip:

`MKV is better for viewing while the pipeline is still writing. MP4 may not be playable until finalized.`
