     `yolov5s_trt11.engine`

     `yolov5s_fp16.engine`
   * Because TensorRT 11 is strongly typed, verify whether the current C++ inference backend assumes FP16 input/output tensors. If it does, the plan must include how to resolve that mismatch:
     Option A: update C++ inference code to support FP32 input/output.
     Option B: add a proper FP16 ONNX conversion/export path and document it clearly.
